# Preset Scale UI

Короткая памятка по следующему этапу работы с пресетами гамм.

## Текущий статус

- Preview-карточка пресета рендерится по `DEFAULT_SCALE_PRESET_ID`.
- Карточка берётся из `data-template="preset-scale-card"`.
- Mount-точка preview: `data-container="preset-scale-preview"`.
- Это временный этап перед модалкой со списком пресетов.

## Что уже зафиксировано

- Основной UI-паттерн для пресета: карточка, не таблица.
- Карточка должна подходить для списка в модальном окне.
- Правая колонка действий остаётся справа.
- Интерфейсные тексты именуются через `label-*`.
- Значения пресета именуются через `preset-*`.
- `comment` не скрывается при пустом значении.
- `hiddenDegrees` показываются списком степеней, а не count.
- `isCustomPreset: false` у встроенных пресетов.
- `isCustomPreset: true` у будущих пользовательских пресетов.
- `apply` доступно всегда.
- `edit` и `remove` доступны только для пользовательских пресетов.

## Важные правила отображения

- `noteCount` считается по реально отображаемым ступеням.
- Число отображаемых ступеней может уменьшаться:
  - через `0` в `intervalPattern`
  - через `hiddenDegrees`
- `familyMoodDisplay` собирается из `mood` и `family`.
- Если есть только одно значение из `mood` / `family`, показывается только оно.
- Если нет ни `mood`, ни `family`, скобки не показываются.

## Поля карточки

Текущая карточка отражает:
- `name`
- `mood` + `family`
- `scaleType`
- `tonic`
- `intervalPattern`
- `contextOffset`
- `modalShift`
- `degreeRotation`
- `hiddenDegrees`
- `comment`

## Техническая опора текущей реализации

- Строковые селекторы preset-card вынесены в `src/constants/index.ts`.
- Для карточки есть типизированные grouped refs в `domRefs.ts`:
  - текстовые элементы
  - label-элементы
  - action buttons
- `renderPresetScalePreview` в `ui.ts` уже разбит на:
  - подготовку текстов карточки
  - рендер текстов
  - рендер label-текстов
  - рендер action buttons

## Что пригодится при создании модалки со списком пресетов

- Использовать существующий `preset-scale-card` как шаблон элемента списка.
- Preview-режим не считать архитектурной нормой: это временная реализация.
- При переходе к модалке логика должна переехать от одной карточки к рендеру массива карточек.
- Если preview станет не нужен, `data-container="preset-scale-preview"` и связанный код можно удалить.

## Что пригодится при назначении action на apply

- Кнопка apply уже имеет `data-control="apply-preset-scale"`.
- `presetScaleId` уже записывается в `data-preset-scale-id` кнопки.
- Применение пресета живёт в `store`, а `ui` только вызывает action по `presetScaleId`.
- После apply `stateActiveScalePresetId` хранит id применённого пресета.
- Любое ручное изменение конфигурации после apply сбрасывает `stateActiveScalePresetId` в `0`.
- Невалидные id пока не обрабатываются: это задача будущего слоя валидации.

## Что пригодится при выпиливании preview

- Удалять нужно не шаблон карточки, а только временную preview-связку.
- Если модалка начнёт использовать тот же template, `preset-scale-card` остаётся.
- Под удаление в первую очередь попадут:
  - `data-container="preset-scale-preview"` в `src/index.html`
  - инициализация preview в `src/app/ui.ts`
  - зависимость от `DEFAULT_SCALE_PRESET_ID` как способа показа карточки на странице
