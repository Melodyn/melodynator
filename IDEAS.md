# Ideas

Облако идей проекта. Не список задач — ориентир для принятия архитектурных решений.
Каждая реализация должна быть минимально достаточной для текущего этапа.

---

## Идентификация собранной гаммы

Пользователь собирает любую гамму в конфигураторе и узнаёт, что получилось. База знаний — деталь реализации: может быть каталогом, AI или чем-то ещё.

- [ ] Текстовый блок на странице конфигуратора: интервалы между ступенями по именам, текущий центр, количество ступеней — бесплатно, без кнопки
- [ ] Идентификация по запросу: пользователь отправляет схему → получает название и объяснение
- [ ] Конструирование по названию: пользователь вводит название → получает схему в конфигураторе
- [ ] База знаний — готовое решение (каталог, AI-API или гибрид). Кастомный код — только обмен данными
- [ ] Формат JSON для обмена с базой знаний определяет приложение, не провайдер
- [ ] Расширенные функции идентификации — платно с пробным лимитом
- [ ] Хроматика (8+ ступеней) — отдельный интерфейс, текущий конфигуратор рассчитан на диатонику

---

## Конфигурация интервальной схемы

Каждая ступень настраивается через выпадающий список с полным диапазоном интервалов. Ядро проекта терпимо к любым комбинациям — интерфейс просто корректно их отображает.

- [x] Выпадающий список на каждую ступень (как у конфигурации ноты струны)
- [x] Значения в списке: 1–12 полутонов с тройной меткой — число и название/я интервала («2 / тон», «3 / малая терция / полудитон»)
- [x] Ячейка конфигуратора показывает только краткое значение: цифру или букву — не название интервала
- [x] Любая комбинация допустима — без валидации на «правильность» гаммы
- [x] Замена текущего подхода с переключением 1/2

> Полные названия интервалов (терция, кварта и т.д.) — в блоке детализации готовой гаммы (см. «Идентификация собранной гаммы»)

## Доступ пользователей

Предусловие для платных функций. Пользователи не должны иметь возможности накопить долг за платные запросы без авторизации.

- [ ] Регистрация и авторизация — готовое решение
- [ ] Лимиты и баланс — привязаны к аккаунту

## Аппликатуры аккордов на гитаре

Chord finder: пользователь выбирает аккорд — видит варианты аппликатуры на грифе в любой позиции. Вероятно, отдельная ветка интерфейса, а не расширение текущего — чтобы не перегружать конфигуратор.

- [ ] Отдельный интерфейс или раздел, не надстройка над текущим грифом
- [ ] Режим грифа: показываются только позиции под пальцы, а не все ноты гаммы
- [ ] Аккорд в любой позиции на грифе, не только в открытой
- [ ] Собственная система именования аккордов
- [ ] Идентификация аккорда по названию — через базу знаний (см. «Идентификация собранной гаммы»)

## Воспроизведение звуков

Клик по ноте играет звук, мультиселект — аккорд. Работает на грифе и клавиатуре. Минимальная реализация — MIDI.

- [ ] Клик по ноте → воспроизведение (гриф и клавиатура)
- [ ] Мультиселект нот → воспроизведение аккордом
- [ ] Реализация — MIDI, без сложного синтеза

## Пресеты гамм и инструментов

Готовые пресеты для быстрого старта и возможность сохранять собственные конфигурации. Бэкенд не нужен — хранение в localStorage, обмен через URL.

- [ ] Встроенные пресеты гамм: мажоры, миноры, лады — определены в коде
- [ ] Встроенные пресеты инструментов: гитара 6/7/12 струн, бас, укулеле — определены в коде
- [ ] Пользовательские пресеты: сохранение текущей конфигурации в localStorage
- [ ] Шаринг: текущее состояние кодируется в query-params → ссылка для передачи
- [ ] По мере роста UI-логики `ui.ts` остаётся координатором, а детальная реализация по зонам выносится в отдельные модули только когда участок становится самостоятельным (например: пресеты, гриф, интервалы)

> `StorageService` — существующая абстракция над хранилищем. Точка расширения для шаринга и пресетов: при добавлении источника (URL-параметры, БД) меняется только она, сторы не трогаются.

## Справка и обучение

Расширенные описания музыкальных концепций — отдельно от основного интерфейса, чтобы не перегружать его. Статика.

- [ ] Короткие подсказки — inline в интерфейсе (tooltips)
- [ ] Расширенные описания — отдельный раздел/страница справки, не спойлеры в основном UI
- [ ] Контент — статические тексты, определены в локализационных файлах
